# Nest Admin 创建实例界面重构设计

## 概述

重新设计 Nest Admin 的创建实例界面，采用标签页分组方式展示预设资源（身份模板、技能、插件），每项以详细信息卡片呈现，包含文件列表。界面视觉风格为简洁专业。

## 设计目标

- 清晰分组：身份模板、技能、插件分标签页展示
- 信息透明：每张卡片显示预设的详细信息和包含文件列表
- 高效操作：支持搜索、已选项管理、GPU/CPU选择

## 标签页内容

### 身份模板标签页

- **卡片内容**：
  - 模板名称
  - 简短描述
  - 包含的配置文件列表（SOUL.md, USER.md, IDENTITY.md, HEARTBEAT.md）
- **交互**：单选模式（每次只选一个身份）

### 技能标签页

- **卡片内容**：
  - 技能名称
  - 简短描述
  - 包含的文件列表（如 SKILL.md）
- **交互**：多选模式，支持搜索过滤

### 插件标签页

- **卡片内容**：
  - 插件名称
  - 简短描述
- **交互**：多选模式

## 视觉规范

### 颜色

- 主题色：`#1890ff`（蓝色）
- 背景色：`#f0f2f5`（页面）、`#ffffff`（卡片）
- 边框色：`#e8e8e8`（默认）、`#1890ff`（选中/hover）
- 已选背景：`#e6f7ff`
- 文字色：`#333`（主）、`#666`（次）、`#999`（辅助）

### 卡片样式

- 背景：白色
- 边框：1px solid #e8e8e8
- 圆角：6px
- 内边距：16px
- Hover：边框变蓝 #1890ff + box-shadow 0 2px 8px rgba(24,144,255,0.15)
- 选中：边框变蓝 + 蓝色勾选图标 + 浅蓝背景

### 标签页样式

- 激活态：蓝色下划线（3px）+ 文字变蓝
- 默认态：灰色文字

### 按钮样式

- 主要按钮：蓝色背景 #1890ff，白色文字
- 次要按钮：白色背景，灰色边框
- 危险按钮：红色文字/边框

## 交互流程

1. 点击"创建实例"按钮，打开弹窗
2. 默认显示"身份模板"标签页
3. 用户可切换标签页浏览不同类别预设
4. 点击卡片可选中/取消选中
5. 已选项目显示在标签页下方的"已选项目"区域
6. 输入实例名称，选择 GPU/CPU 镜像
7. 点击"创建实例"提交表单

## API 需求

### 获取预设详情

现有 API 可复用：
- `GET /api/presets/identities` - 身份模板列表
- `GET /api/presets/skills` - 技能列表
- `GET /api/presets/plugins` - 插件列表

### 新增 API（如需要）

- `GET /api/presets/identities/{id}` - 获取身份模板详情（含文件列表）
- `GET /api/presets/skills/{id}` - 获取技能详情（含文件列表）

## 前端实现要点

- 预设数据加载：页面加载时一次性获取所有预设
- 搜索过滤：前端实时过滤，支持按名称/描述搜索
- 状态管理：维护 selectedIdentity, selectedSkills, selectedPlugins 三个状态
- 标签页切换：根据当前标签页显示对应类别的已选项
- 表单验证：创建时验证实例名称非空、至少选择一个身份（如需要）

## 后端实现要点

- `apply_identity_to_instance()`: 复制身份模板文件到实例 workspace 目录
- `apply_skills_to_instance()`: 复制技能目录到实例 workspace/skills 目录
- `apply_plugins_to_instance()`: 复制插件到实例 skills 和 extensions 目录

## 文件变更

- `nest-admin/frontend/index.html` - 重构创建实例弹窗 UI
- `nest-admin/backend/main.py` - 如需要新增获取预设详情的 API
